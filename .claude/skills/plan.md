# Plan Skill

You are creating an implementation plan for a feature or change in the **SunbirdEd Mobile App** (Ionic React 8 + Capacitor 7 + TypeScript).

To get started, please provide:

1. **The feature or change you want to build** — what functionality should be added or modified?
2. **Where it should live** — is this a new page, a new player, a hook, a service, or a native change?
3. **Any specific requirements** — user flows, API details, offline behaviour, platform-specific needs, or constraints?

If the request is clear, I'll then:
- Explore the codebase (pages, components, hooks, services, existing patterns)
- Produce a step-by-step implementation plan
- Ask for your approval before any implementation begins

## Plan Output Format

### Summary
What is being built and why.

### Files to Create
List each new file with its purpose.

### Files to Modify
List each changed file with what changes and why.

### Implementation Steps
Ordered, actionable steps. Each step should be small enough to verify independently:
1. Step with specific file and change description
2. ...

### Verification
- How to test the feature manually on Android/iOS
- Which automated tests to run: `npx vitest run src/path/to/test.tsx`
- Any Capacitor sync needed: `npx cap sync`

### Risks & Dependencies
Any concerns about native compatibility, package versions, API availability, or migration needs.
